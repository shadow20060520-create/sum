# **汉语教学语音纠错辅助工具 \- 项目文件结构 (V1.0)**

本项目采用 Monorepo 风格的目录结构进行管理，物理隔离后端/教师端（Next.js）与学生端（微信小程序），并独立出数据库版本控制。

## **工程架构设计说明 (Design Decisions)**

1. **supabase/migrations 独立管理**：强力建议使用 Supabase CLI 本地开发，所有的建表 (schema.sql) 和权限 (rls.sql) 作为 migration 脚本。这样团队可以利用版本控制随时重建完整的测试数据库，符合现代 DevOps 规范。  
2. **web-platform/src/app/api 层级**：这里是安全命脉。前端严禁直接进行敏感的写操作，必须调用这里的 API。这里的接口会使用 supabase\_admin\_key (Service Role) 穿透 RLS 进行数据验证和写入。  
3. **miniprogram/utils/iflytek-ws.js**：由于 Vercel Serverless 不支持 WebSocket 服务器，该工具类负责向 /api/iflytek/auth 索要短时效签名，然后直接通过 wss:// 连接科大讯飞，实现录音毫秒级流式反馈，解决 Vercel 平台的架构短板。
